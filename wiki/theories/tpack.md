# TPACK (Technological Pedagogical Content Knowledge)

**요약**: 교사가 특정 내용을 특정 학습자에게 가르치기 위해 기술을 효과적으로 통합하려면, 기술 지식(TK)·교육학 지식(PK)·내용 지식(CK) 세 가지의 교차 영역에 대한 이해가 필요하다는 프레임워크.

---

## 이론적 구조

Mishra & Koehler(2006)는 Shulman(1986)의 PCK(Pedagogical Content Knowledge)를 확장하여 기술 차원을 추가했다.

```
        CK (내용 지식)
       /              \
      /                \
  PCK                  TCK
     \                /
      \              /
  PK (교육학 지식) -- TPACK -- TK (기술 지식)
           \                 /
            \               /
               TPK
```

### 7가지 지식 영역

| 약어 | 명칭 | 내용 |
|---|---|---|
| CK | Content Knowledge | 교과 내용 전문 지식 |
| PK | Pedagogical Knowledge | 일반 교수·학습 방법 지식 |
| TK | Technological Knowledge | 기술 도구 사용 지식 |
| PCK | Pedagogical Content Knowledge | 특정 내용을 가르치는 방법 |
| TCK | Technological Content Knowledge | 기술로 내용을 표현하는 방법 |
| TPK | Technological Pedagogical Knowledge | 기술을 교수 전략에 활용하는 방법 |
| **TPACK** | **통합 지식** | **세 영역의 통합적 이해** |

## TPACK 측정

주요 측정 방식:
- **자기보고 설문**: 가장 일반적, 편향 존재
- **수행 기반 평가**: 실제 수업 계획서·수업 관찰
- **개방형 응답 분석**: TPACK 수준의 질적 평가

## 비판 및 한계

- 7개 지식 영역의 구분이 실제로는 모호
- 맥락(context) 지식의 불충분한 반영
- 기술 변화 속도 → TK 정의의 지속적 갱신 필요
- 교사 지식의 **측정 타당도** 문제
